## 执行顺序

1. **F-01 — P0**：初始化 Flutter 工程 → 搭建分层目录 → core:common → core:database
2. **F-02 — P0**：core:reminder → core:notification → feature:home → 跑通手动录入全流程
3. **F-03 — P0**：集成 ASR/LLM 模型 → core:voice → feature:voice_input → 跑通语音录入全流程
4. **F-04 — P1**：feature:group_manage → feature:cleanup → 补货提醒
5. **F-05 — P1**：全量联调 → 补测试 → 机型适配 → 模型下载优化 → 打包上架

**并行策略**：F-02、F-03、F-04 都仅依赖 F-01，三者之间无依赖。F-01 完成后，这三个功能可以依次或并行推进（WIP=1 约束下，需顺序进行，但规划上互不阻塞）。
