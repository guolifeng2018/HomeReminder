> 本文件的每一条都是硬约束。agent 违反任一条 → reviewer L1 直接 FAIL。

## 架构约束

1. **必须**遵守分层依赖规则：feature → core，依赖方向不可逆。下层禁止 import 上层模块。
2. **必须**使用 Riverpod 做状态管理，禁止引入其他状态管理框架（BLoC、GetX 等）。
3. **禁止**在 Widget 中直接访问 Drift 数据库，必须通过 Repository/Service 层。
4. **必须**使用 Method Channel 桥接 ASR/LLM 原生 SDK，禁止在 Dart 层直接加载推理引擎。

---

## 代码规范

1. **禁止**任何形式的网络请求、数据上传、日志上报。所有数据全程本地处理。
2. **必须**按需申请权限（麦克风、通知、存储），权限说明文案仅标注对应真实用途，禁止多余权限声明。
3. **必须**对 Android 10+ / iOS 15+ 做平台 API 版本兼容检查，禁止使用超出最低版本的 API 不做 fallback。

---

## 测试约束

1. **必须**为所有数据库 CRUD 操作编写单元测试，覆盖增删改查主路径。
2. **必须**为时间解析逻辑（含「下周」「半个月后」等口语时间）编写测试用例覆盖。

---

## 工具链约束

1. **禁止**将 ASR/LLM 模型文件（SenseVoice-Tiny、Qwen-140M GGUF）打包进安装包。
2. **必须**使用 `flutter analyze` 零报错作为 L1 门禁，禁止 warnings 遗留。
3. **禁止**提交包含调试代码（`print` 调试输出、`debugger` 断点、TODO 注释）的代码。
